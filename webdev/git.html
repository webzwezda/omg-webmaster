Настройка git для первого использования

Основные моменты рабыты в консольной версии гита из под ubuntu.

Установка

Все начинается именно с нее, а иногда так зарождается жизнь.

sudo apt install git-core

Инициализация пользователя

Обязательное действие без которого не чего не заработает

git config --global user.name "Ilya"
git config --global user.email "dadast@outlook.com"

Просмотреть настройки

git config --lis
git config --global --list //Вывод глобальных настроек

Команды для работы с репозиторием

Работа с репозиторием подразумевает удаление и комиты файлов

Удаление

Удалит файл из гита, локально он останется на месте

git rm --cached file.txt

Просмотреть лог

git log //логи коммитов
git log -2 //покажет последние два коммита
git log -p // покажет нам изменения между коммитами

Удаление из staging area случайно добавленного файла

git add index.html //Типо случайно добавили файл
git reset HEAD index.html //Типо случайно удалили из staging area

Отправка на GitHub

git remote add origin [rep-name]
git remote rm origin [rep-name] //Удалит
git remote set-url origin [rep-name] //Переназначить
git push -u origin master //Отправка в  origin

Ключ -u говорит о том что бы гит запомнил куда он пушит что позволит нам ипользовать сокращенные команды git push и git pull, система контроля версий вспомнит какие данные должны бить дописанны нами.

Ветки

git branch //покажет ветки
git branch [NAME] //создаст ветку
git checkout [NAME] //перейти в
git merge [NAME] //Добавить в себя ветку
git branch -d [NAME] //удалить ветку
git checkout -b gh-pages // -b создаст и перейдет

Shh

Создадим два разных ключа для.

ssh-keygen -t rsa -f ~/.ssh/key1 -C "other-account@example.com"
ssh-keygen -t rsa -f ~/.ssh/key2 -C "other-account@example.com"

Создадим конфиг в ~/.ssh/config

host gh-acc1
hostname github.com
user git
IdentityFile ~/.ssh/id_rsa
IdentitiesOnly yes

host gh-acc2
hostname github.com
user git
IdentityFile ~/.ssh/key2
IdentitiesOnly yes

Присвоим нашему названию куда он будет пушить

git remote add other git@gh-acc2:user/project.git

Git and Kdiff3

Это дополнительный инструмет для разрешения конфликтов при смерживании

Установка

В Linux не будет проблем

sudo apt-get update
sudo apt-get install kdiff3

Настройка конфига гита

Без этого не заработает. Инфа взята от сюда.

git config --global merge.tool kdiff3

git config --global --add merge.tool kdiff3
git config --global --add mergetool.kdiff3.path "/usr/bin/kdiff3"
git config --global --add mergetool.kdiff3.trustExitCode false
git config --global --add mergetool.prompt false

git config --global --add diff.guitool kdiff3
git config --global --add difftool.kdiff3.path "/usr/bin/kdiff3"
git config --global --add difftool.kdiff3.trustExitCode false
git config --global --add difftool.prompt false

Пример работы

Втянем в себя ветку master

git merge master

Увидив конфликт мы разрешаем его при помощи нашей утилиты

git mergetool
